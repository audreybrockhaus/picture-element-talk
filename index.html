The problem with pictures on responsive webpages

  • Raster image files (jpg, png, gif, etc.) have a fixed size (in pixels).
  • Responsive web pages need to display content in a variety of sizes.

Enter the picture element!

"The HTML <picture> element is a container used to specify multiple <source> elements for a specific <img> contained in it. The browser will choose the most suitable source according to the current layout of the page (the constraints of the box the image will appear in) and the device it will be displayed on (e.g. a normal or hiDPI device.)"

— Mozilla Developer Network

Features & Benefits;

  • Allows the browser to pre-load the correct images before css is loaded, optimized for screen size and resolution.
  • Can easily load entirely different images for different screen sizes (which is called art direction).
  • Can load exotic image formats like WebP and svg only in browsers that support them.
    • The picture element, or the srcset attribute on an image tag, can do a whole ton of stuff, including loading different images for different screens based on the device's pixel density, load fancy image formats in browsers that support them, and more. It's really generally just a way to give the browser some options about which image source is right for it. So I don't get in over my head here I'm just going to be talking about the most simple use case for it, loading different images for different screen sizes.

The Markup
The html is structured similarly to the html5 video element, with a wrapping "picture" element and then a list of sources that the browser will choose from.

Browser Support

Picturefill (Picture polyfill)

http://scottjehl.github.io/picturefill/

(works beautifully all the way back to IE8)

The picturefill polyfill is basically THE picture polyfill, I have never seen any other one recommended or even mentioned, they kind of own this. It was made by Filament group, which have made all kinds of useful things including Grunticon, which is a kind-of-a-big-deal tool for svg icons with fallbacks, and Respond, which is a polyfill for media queries in IE8, which was used on RCI here at melt.
